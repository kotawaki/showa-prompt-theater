# エピソード原稿の追加ルール

このフォルダには公開前の本編原稿をMarkdownで保存します。現時点では本編エピソードはありません。

## ファイル名

`episode-001.md`、`episode-002.md` のように3桁の連番を使用します。

## 原稿テンプレート

```markdown
# 第XXX話「タイトル」

## 1ページ目：起

<!-- 数行中心。今回の仕事や問題を提示する。 -->

## 2ページ目：承

<!-- 曖昧な指示によるズレを広げる。 -->

## 3ページ目：転

<!-- 発見、誤解、予想外の行動で状況を動かす。 -->

## 4ページ目：結

<!-- 小さな解決と余韻を置く。 -->

## 5ページ目：今日の昭和プロンプト・用語解説

### 今日の昭和プロンプト

「」

### ポンコツ上司が覚えた用語

**用語名：**

説明。

ポンコツ上司いわく「」

### おまけの掛け合い

**ポンコツ上司：**

「」

**AI部下：**

「」
```

## 追加時のチェック

- 5ページ構成になっている。
- 各ページが数行中心で、余白がある。
- 1話につき学びを一つに絞っている。
- キャラクターの口調と成長段階が設定に合っている。
- プロンプトと用語を各台帳にも追記している。
- 実在サービス名、規約回避、危険な自動化手順を書いていない。
- 公開用HTMLを `docs/episodes/` に追加し、トップページからリンクしている。

## 第1話を追加する場合

1. このテンプレートから `episode-001.md` を作成する。
2. 第1話では、二人の初対面と「曖昧な指示が通じない」段階を守る。
3. パチンコ分析はまだ始めず、会社から与えられた業務を題材にする。
4. 5ページ目で扱うプロンプト技術を一つだけ決める。
5. 原稿完成後に `../prompts/showa_prompts.md` と `../glossary/terms.md` を更新する。
6. 同じ内容を `../docs/episodes/episode-001.html` として公開用に整える。

画像や4コマ版を後から追加しても、Markdown原稿は文章版の正本として残します。画像だけに存在するせりふや重要情報を作らないようにしてください。
