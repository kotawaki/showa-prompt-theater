# 執筆ルール

## 1話の基本構成

1話は必ず6ページで構成する。

1. タイトルページ：話数とタイトルだけを置く章扉
2. 起：今回の仕事や問題が始まる
3. 承：曖昧な指示によるズレが広がる
4. 転：誤解、発見、予想外の行動で状況が動く
5. 結：小さな解決と次につながる余韻
6. 学び：今日の昭和プロンプト、用語解説、おまけの掛け合い

## 文章量とテンポ

- 4コマ漫画のように、短い発言と間で進める。
- 各ページは数行を中心とし、空白を多く取る。
- 長い説明文や、一段落に情報を詰め込む書き方を避ける。
- スマホで一目ずつ追える長さを優先する。
- 「原稿用紙を埋め切れない小学生の作文」ほど余白があってよい。
- 1ページには基本的に一つの笑い、発見、感情の動きだけを置く。
- タイトルページは章扉として文字数を抑え、中央付近に縦書きで配置する。
- ページ内スクロールに頼らず、本文が収まらない場合は文章を短くするか次ページへ分ける。

## 会話と口調

- ポンコツ上司は昭和的な勢いと関西寄りのくだけた口調で話す。
- AI部下は簡潔で冷静。曖昧さを具体的に指摘する。
- AI部下の正論だけで話を終えず、上司の人間味や意外な着眼点も残す。
- 回を重ねるごとに、二人の言葉が少しずつ通じる変化を描く。
- キャラクターを単純な無能役、万能役に固定しない。

## 6ページ目の必須項目

```text
今日の昭和プロンプト：
「ここに、その話で学んだ再利用可能な指示を書く」

ポンコツ上司が覚えた用語：
用語名：
短く平易な説明。
ポンコツ上司なりの言い換え。

おまけの掛け合い：
ポンコツ上司：
「短いひと言」

AI部下：
「短い返答」
```

## 公開時の表示方針

- 本文には `writing-mode: vertical-rl` を使用する。
- 右から左へ読む文庫本風の雰囲気を作る。
- 文字を大きめにし、行間と余白を広く取る。
- ページ単位をカードや境界線で明確に区切る。
- スマホでは横方向にページを送れる表示を基本とする。
- 6ページ目の学び部分は、読みやすさを優先して横書きでもよい。
- 本文が途中で切れないことを演出より優先し、各ページ内ではスクロールさせない。
- 将来の画像用に `.character-visual`、`.comic-panel`、`.episode-illustration` を使用する。

## 安全性とフィクション

- 実在サービス名は使用しない。
- 規約回避や不正利用を助ける具体的手順は書かない。
- 危険な自動化、賭け事の必勝法、個人情報を扱う手順は具体化しない。
- 必要な題材は架空の会社、架空の仕組み、ぼかした表現へ置き換える。
- パチンコ描写は娯楽上のフィクションとし、勝利を保証する表現を避ける。
